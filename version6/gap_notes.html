Supplementary Gap Notes

This note documents reconstruction gaps that remain after aligning the paper to the original report. For each gap, it states exactly what artifact or analysis is needed to resolve the issue in a submission-ready version. Unless explicitly stated otherwise, all statistical quantities discussed here are assumed to be recoverable from the saved original experiment outputs, meaning the archived LLM requests and responses, parsed answers, profile assignments, and evaluation tables, without issuing any new LLM calls.

1. Skill-source reconstruction gap

What is directly supported by the original report: the report states that MathCAMPS items are tagged according to U.S. Common Core standards; each educational standard is mapped to a Domain; and each Domain is treated as a distinct Skill for the purposes of controllable simulation.

What could not be recovered from the current workspace: an explicit grade-by-grade table of formal skill labels, standard IDs, or a machine-readable mapping file linking item IDs to exact skill names.

What must be added to resolve this gap:

  1. Export the original item-to-skill mapping from the benchmark source files, ideally as CSV or JSON with columns such as grade, item_id, standard_id, domain, and skill_label.
  2. Add a paper appendix table listing the exact skill labels used in each reported grade, at minimum for Grades 4 and 5.
  3. Replace any representative or reconstructed skill-family tables in the main text with either exact archived labels or a clearly marked contextual table plus a link to the archival mapping file.
  4. Archive the mapping file together with the benchmark release so that readers can reproduce the aggregation from items to skills.

Paper sections affected: Section 4.2, Table 1, Figure 1 terminology, and any passage that refers to domain-level skill labels.

Best-effort hypothesis, not verified: the most likely skill source used in the original experiments is a grade-specific Common Core mathematics domain mapping extracted from MathCAMPS. Under this interpretation, the operational skill labels were probably domain-level curriculum labels rather than ad hoc manual skill names.
Grade Likely domain-level skill source Status

The paper therefore retains the report's supported claim that the skill taxonomy derives from Common Core standard-to-domain mappings, while avoiding unsupported reconstruction of exact formal labels. The gap is fully resolved only once the archived mapping file is restored and cited directly.

1A. Example-item documentation gap

What appears in the main paper: Section 4.2 and Table 2 present example item annotations and open-to-multiple-choice conversions as if they were part of the benchmark documentation.

What should be done to resolve this gap:

  1. Replace the current example rows with exact archived benchmark items, preserving original item wording, assigned skill label, and final multiple-choice options.
  2. If the exact released items cannot be disclosed, mark the table as synthetic in an appendix and move exact benchmark examples to a restricted artifact package.
  3. Add a short appendix note describing the distractor-construction policy used to convert open items into multiple-choice items.

Paper sections affected: Section 4.2.2 and Table 2.

2. Post hoc statistical-analysis gap

The current main paper does not include the strongest secondary statistical analyses that could be derived from the archived experiment outputs. Under the assumption that all original LLM requests and responses were saved, the analyses below can be computed exactly from the existing experimental data without issuing any additional LLM calls.

Analysis Why it helps What should be computed from archived outputs Recommended placement in the paper What should be added to the paper
Profile-matched RMSE differences between prompting strategies Quantifies how much better the combined prompt is than each ablation using the preserved profile-level outputs. For each profile and grade, compute paired RMSE differences between combined and each ablation; then report mean difference, median difference, and exact bootstrap or permutation interval. Best placement: Section 6.2, immediately after Figure 4, or as an appendix mini-figure. Add exact paired contrasts between Combined and each ablation as a compact figure or table derived from archived outputs.
Retained-forgotten accuracy gap by model Makes the separation between successful retention and successful forgetting easier to interpret than forgotten-skill accuracy alone. For each model, compute retained-skill mean accuracy minus forgotten-skill mean accuracy over the preserved profile-level outputs, together with exact uncertainty intervals. Best placement: Section 6.3, immediately after Figure 5 and Table 4. Add exact model-wise retained-minus-forgotten gaps and intervals derived from archived outputs.
Variance decomposition over preserved outputs Separates the contribution of model, prompt, grade, and skill status to overall controllability variation. Fit a post hoc ANOVA-style or mixed-effects decomposition using archived per-profile outputs, with factors for model, prompt strategy, grade, and skill status. Best placement: Section 7.1 or 7.2, where interpretation of model and prompt effects is discussed. Secondary option: appendix only. Add one interpretive paragraph in Discussion supported by an appendix table reporting factor contributions or partial effect sizes.
Cross-skill influence heatmap Directly operationalizes the cross-skill influence definition and makes collateral effects visible rather than inferred from correlations alone. Compute the full matrix of skill-to-skill influence values using archived per-skill accuracy under each suppression profile, then aggregate by grade and optionally by model. Best placement: Section 6.4, immediately after Figure 6. Add a new heatmap figure showing direct effects on the diagonal and collateral effects off the diagonal, plus one paragraph interpreting whether suppression remains localized.
Profile-level score distribution figure Shows whether controllability is consistent across profiles or driven by a small subset of easy cases. For each model and prompt strategy, plot the distribution of profile-level RMSE or controllability scores across all enumerated profiles. Best placement: Section 6.2 or 6.3 as an auxiliary figure. Add violin plots, box plots, or ridge plots to show spread, skew, and overlap across strategies and models.
None of the additions above should require re-querying any model if the original requests, responses, and parsed evaluation tables were preserved. They are post hoc analyses over existing experimental data only.

Recommended computation workflow for these additions:

  1. Load the archived per-profile evaluation table with columns for model, grade, prompt_strategy, profile_id, skill_id, target_state, and accuracy.
  2. Compute profile-level RMSE for each model, grade, prompt strategy, and profile.
  3. For prompt comparisons, pair profiles within each grade and model, then compute Combined minus Few-Shot and Combined minus Rule-Based RMSE differences.
  4. For retained-forgotten gaps, aggregate accuracy separately over skills with target_state = 1 and target_state = 0, then subtract forgotten from retained within each model.
  5. For cross-skill influence, use suppression profiles to compute per-skill changes relative to the perfect-student baseline and aggregate into influence matrices.
  6. For variance decomposition, fit a post hoc linear or mixed-effects model using archived outputs, with factors for model, prompt strategy, grade, and skill status.
  7. Write the exact results into the recommended sections or appendix figures, and store the full computation output in the artifact bundle or appendix tables.

Minimum archived data required for exact recomputation:

  • Prompt strategy used for each interaction
  • Model identifier for each interaction
  • Grade, profile ID, and target skill vector for each simulated student
  • Raw model request and raw model response, or a faithful archived parsed answer for each item
  • Item-to-skill mapping used during scoring
  • Per-item correctness after answer parsing

If these six components exist in the archived experiment data, then the prompt-comparison contrasts, retained-forgotten gaps, cross-skill influence summaries, and variance-decomposition analyses are all recoverable exactly from the original experiment outputs.

3. Reproducibility artifact gap

What is missing: the paper does not link an artifact bundle containing exact prompts, few-shot examples, scoring scripts, model identifiers, or the archived evaluation outputs used to generate the figures.

What must be added to resolve this gap:

  1. Create a public artifact package or supplement containing the exact rule-based, few-shot, and combined prompt templates.
  2. Record exact model identifiers, API provider names, access dates, decoding parameters, and any parsing rules for invalid outputs.
  3. Release archived evaluation tables at the profile level or, at minimum, provide model × grade × skill summary tables for all reported results.
  4. Add a stable repository or DOI link in the paper's reproducibility section.

Paper sections affected: Sections 5.3, 7.5, 8, and the end-of-paper technical-note link.

4. Benchmark-validity gap

What is missing: the current paper states that some items were replaced after repeated cross-model errors, but it does not provide an audit trail describing which items were replaced, why they were replaced, and which criteria were applied independently of the experimental hypotheses.

What must be added to resolve this gap:

  1. Create an item-audit table with columns such as item_id, grade, original_skill, replacement_reason, and replacement_item_id.
  2. State whether replacement decisions were made before or after the final experimental comparisons were examined.
  3. Move this audit table to an appendix or artifact package and reference it explicitly in the main text.

Paper sections affected: Section 4.2.3 and Section 7.4.
